# Спецификация и архитектура транслятора TuringCode

## 1. Введение
Проект представляет собой реализацию Тьюринг-полного компилятора, осуществляющего трансляцию высокоуровневого императивного псевдокода в набор микроинструкций (состояний и правил перехода) для классической одноленточной Машины Тьюринга. Вычислительная модель базируется на унарной системе счисления и статической аллокации памяти с заданными защитными интервалами.

## 2. Архитектура конвейера трансляции
Процесс обработки и выполнения исходного кода разделен на несколько строго детерминированных логических этапов.

### 2.1. Лексический анализ (Scanner)
На первом этапе подсистема `Compiler` осуществляет лексический разбор исходного текста, разбивая его на неделимые лексемы (токены). Алгоритм лексера способен распознавать ключевые слова типов данных (`bool`, `byte`, `int`), операторы арифметики и ветвления (`+`, `-`, `=`, `<`, `>`), инкремент и декремент (`++`, `--`), а также структурные разделители (скобки) независимо от наличия пробельных символов.

### 2.2. Распределение памяти (Static Memory Allocation)
Выделение физического адресного пространства на ленте производится статически до этапа кодогенерации. Транслятор реализует строгую систему типизации, которая определяет размер защитного буфера (Padding) для предотвращения уязвимости переполнения буфера (Buffer Overflow) в процессе унарного сложения:
* `bool`: выделяет 1 ячейку памяти.
* `byte`: выделяет 255 ячеек резерва.
* `int`: выделяет 10 000 ячеек резерва.

Числовые константы, встречающиеся в коде, инициализируются как скрытые Read-Only переменные (например, `_c0`) с нулевым буфером расширения, что оптимизирует общую длину ленты. На физическом уровне переменная формируется в формате `#<адрес>:<унарное_значение>#<защитный_буфер>`.

### 2.3. Синтаксически управляемая трансляция (AST и State Routing)
После удаления токенов типов из лексического потока, транслятор переходит к этапу генерации направленного графа состояний.
* **Арифметические операции:** Транслируются в вызовы предкомпилированных макрокоманд (инкремент, сложение, усеченное вычитание, умножение). Каждая макрокоманда гарантирует возврат каретки к абсолютному маркеру начала ленты (`^`), обеспечивая безопасную конкатенацию инструкций. Для избежания коллизий (самоприсваивания при умножении) компилятор автоматически задействует системный регистр АЛУ `_temp`.
* **Поток управления (Control Flow):** Обработка вложенных структур `while`, `for`, `if/else` реализуется посредством стека контекстов (`std::vector<BlockScope>`). Компилятор генерирует уникальные узлы графа (`condState`, `bodyState`, `endState`) и использует макрос `GenerateCompare` для маршрутизации автомата.

### 2.4. Система прерываний и выполнение (Execution & Syscalls)
Исполнение полученного микрокода управляется методом `Execute()`, функционирующим как гипервизор. При переходе Машины Тьюринга в состояние, зарегистрированное как системный вызов (`printHooks_`), выполнение приостанавливается. Подсистема декодирует унарное представление запрашиваемой переменной в десятичный формат, осуществляет вывод в `stdout` и возобновляет симуляцию.

## 3. Синтаксис и правила разработки программ

Исходный код алгоритмов должен быть помещен в текстовый файл `program.txt` в корневом каталоге проекта.

**Спецификация синтаксиса:**
* **Инициализация:** Обязательная явная типизация при объявлении новой переменной (`bool`, `byte`, `int`).
* **Операторы:** Поддерживаются `=` (присваивание), `+` (сложение), `-` (усеченное вычитание), `*` (умножение), `++` (инкремент), `--` (декремент).
* **Ветвление:** Условные конструкции `if` и каскадные `else if` / `else`. В качестве условия ожидается идентификатор переменной (истина эквивалентна значению, строго большему нуля).
* **Циклы:** Структура `while` (выполнение до обнуления переменной-условия) и полная структура `for` (включающая блоки инициализации, проверки отношения `<` или `>`, и арифметического шага).
* **Ввод/Вывод:** Инструкция `print <имя_переменной>` осуществляет системный вызов для трассировки значений.

**Пример алгоритма (вычисление факториала):**
```text
int fact = 1

for ( byte i = 1 ; i < 6 ; i++ ) {
  fact = fact * i
  print fact
}
```

## 4. Сборка и интеграция

Программный комплекс требует компилятора, поддерживающего стандарт C++17. 

1. Разместите файл `program.txt` с написанным алгоритмом в директории исходных кодов.
2. Выполните компиляцию проекта с использованием `make`:
   ```bash
   make clean
   make
   ```
3. Запустите скомпилированный бинарный файл:
   ```bash
   ./turing_machine
   ```

После запуска `main.cpp` инициализирует Машину Тьюринга, делегирует компилятору построение дерева состояний на основе файла `program.txt`, разворачивает память на ленте и запускает процесс исполнения. Результаты выполнения системных прерываний будут последовательно выводиться в стандартный поток.
